# 검색 결과 탭 분리 구현 계획

## 목표

- 검색 결과를 **질문/공부** 두 탭으로 분리.
- 탭 전환 시 해당 카테고리 결과만 보여줌.
- 결과 개수와 빈 상태 메시지를 각 탭에 맞게 제공.

## 범위

- `/search` 페이지에 탭 UI 추가.
- 클라이언트 필터링 방식 유지 (정적 배포 호환).

## 변경 대상

- `src/app/search/page.tsx`
- `src/components/SearchClient.tsx`

## 설계 개요

### 1) 데이터 구조

- 서버에서 `questions`, `studies`를 분리해서 내려줌.
- 클라이언트에서 `activeTab` 상태로 분리 렌더링.

### 2) UI 흐름

- 상단 탭 버튼:
  - `질문` / `공부`
  - 활성 탭 강조(underline 또는 배경)
- 탭별 결과 리스트:
  - 활성 탭 데이터만 `DocumentList`에 전달
  - 결과 없으면 탭 전용 메시지 표시

### 3) 검색 필터

- 입력값 기준으로 **제목/키워드** 필터
- 필터는 두 카테고리에 **동일 기준**으로 적용

## 구현 단계

1. `SearchClient`에 탭 상태 추가 (`"questions" | "studies"`).
2. props를 `{ questions, studies }`로 변경.
3. 필터 결과를 카테고리별로 계산.
4. 탭 UI 추가 및 탭별 `DocumentList` 렌더링.
5. 빈 상태 메시지를 탭별로 분리.

## 검증 기준

- 탭 전환 시 해당 카테고리 결과만 표시.
- 검색어 입력 시 두 탭 결과가 각각 필터됨.
- 결과 없을 때 탭별 메시지가 표시됨.

## 리스크/대응

- 결과가 많아질 경우 렌더링 비용 증가 → 검색어 입력 시 debounce 고려.
