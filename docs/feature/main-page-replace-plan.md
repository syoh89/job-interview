## 개요

- 메인 페이지를 StarryActivityShowcase/Calendar 중심으로 완전 대체한다.
- 키워드 탐색은 `/keywords` 페이지로 이동해 키워드 목록 + 검색을 통합한다.
- 메인은 브랜드 인상/활동 지표 중심, 탐색은 별도 허브로 분리한다.

## 요구사항 정리

- 메인 페이지: 제공된 이미지 톤과 유사한 카드 레이아웃 중심
- 키워드 섹션: 메인에서 제거
- `/keywords`: 키워드 목록 + 검색 기능 결합

## 현재 구조 점검 요약

- 메인: `src/app/page.tsx`
- 키워드 통합: `src/app/keywords/page.tsx`
- 키워드 상세: `src/app/keywords/[keyword]/page.tsx`

## 구현 단계

### 1단계: 메인 페이지 완전 대체

- 메인에는 StarryActivityShowcase/Calendar 섹션만 유지
- 키워드 관련 섹션 제거
- 카드 스타일/타이포는 토큰 기반으로 통일

### 2단계: `/keywords` 통합 페이지 구성

- 키워드 목록(기술/일반) + 검색 UI를 한 페이지에서 제공
- 검색은 기존 `SearchClient` 재사용
- 결과 목록은 `DocumentList` 활용

### 3단계: 탐색 흐름 정리

- `Navigation`에 `/keywords` 링크 고정
- `/search` 페이지 제거 및 탐색 허브를 `/keywords`로 통일

## 검증 체크리스트

- 메인에서 키워드 섹션이 완전히 제거되었는지 확인
- `/keywords` 페이지에서 키워드 목록과 검색이 정상 작동하는지 확인
- `/keywords/[keyword]` 상세 페이지로 이동이 유지되는지 확인
- 검색 입력 시 질문/공부 탭이 정상 필터링되는지 확인

## 아이디어 코멘트

- 메인 완전 대체는 브랜드 인상과 집중도를 높인다.
- 키워드/검색 통합은 탐색 맥락이 분명해져 사용자 이해도가 높아진다.
- 탐색 허브(`/keywords`)가 중심이 되므로, 향후 키워드 정렬/필터/즐겨찾기 확장에 유리하다.

## 추후 확장 아이디어

- 키워드 정렬(빈도/최근 업데이트)
- 검색 결과 하이라이트(키워드 강조)
- 키워드 즐겨찾기/최근 사용
